# 需求说明书

## 1. 业务场景定义
本项目为一个多功能智能助手平台，包含以下核心能力：
- **智能问答（Retrieval QA）**：基于私有技术文档的精准知识检索与增强问答（RAG），支持文档来源溯源。
- **SQL生成与执行（Structured SQL Agent）**：通过自然语言交互，理解数据库表结构，生成并执行 SQL 进行数据查询，包含错误自愈和重试机制。
- **多轮会话记忆**：跨越对话周期的状态保持能力，支持长文本和滑动窗口截断，确保模型不丢失上下文。

## 2. 功能范围边界
- **核心链路**：包含基于文档的 QA 链及操作 Postgres 数据库的 SQL Agent 链。
- **用户交互**：提供 Web 界面，支持 Markdown 渲染、流式响应（SSE）、代码高亮与溯源展示。
- **数据流转**：通过可重复执行的 ETL 管道导入文档到向量数据库，并记录分块及其来源。
- **运维与监控**：提供 Prometheus 的监控指标导出，实现 Grafana 可视化。

## 3. 非功能性指标
- **单次推理延迟**：≤2 s，90th 百分位（基于 GPT-3.5-turbo）。
- **并发性能**：支持并发 ≥20 QPS，并基于 Redis 提供单 IP 30 req/min 限流。
- **事实性准确率**：答案准确率 ≥90 %（以人工标注 200 条测试集为基准）。
- **可用性**：后端系统具备探活机制 `/health`。
